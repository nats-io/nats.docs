# Durable

Если приложение хочет продолжить потребление сообщений с места остановки, ему нужна durable-подписка. Для этого передается durable-имя, которое комбинируется с client ID, указанным при создании подключения. Сервер сохраняет состояние такой подписки даже после закрытия клиентского подключения.

_**Примечание: стартовая позиция, переданная клиентом при перезапуске durable-подписки, игнорируется.**_

Когда приложению нужно перестать получать сообщения по durable-подписке, следует закрыть подписку, но _не выполнять unsubscribe_. Если в клиентской библиотеке нет возможности закрытия подписки, приложение должно закрыть подключение.

Когда приложению нужно удалить подписку, нужно выполнить unsubscribe. После unsubscribe состояние удаляется, и durable-имя можно использовать снова, но это уже будет новая durable-подписка со стартовой позицией, указанной клиентом при ее создании.
