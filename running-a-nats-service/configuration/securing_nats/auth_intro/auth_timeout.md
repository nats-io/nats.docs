# Таймаут аутентификации

Можно задать таймаут, чтобы ограничить время ожидания аутентификации клиента сервером.

Если значение не указано или указано "0", по умолчанию используется значение на 1 секунду больше, чем `tls_timeout`.

Если указано некорректное значение, будет использовано значение по умолчанию 1 секунда.

Если клиент не аутентифицируется за отведенное время, сервер разрывает соединение, чтобы предотвратить злоупотребления.

Таймауты задаются в секундах (в том числе дробных).
В отличие от `tls_timeout`, нельзя использовать «читаемые» значения вроде `10s` —
нужно указать число, которое будет интерпретировано как секунды.
`10` — это 10 секунд, `3.5` — это 3 секунды и 500 миллисекунд и т. д.

Как и с TLS‑таймаутами, слишком большие таймауты могут быть возможностью для злоупотреблений. При настройке таймаута аутентификации важно помнить, что он должен быть больше, чем `tls timeout`, так как таймаут аутентификации включает время TLS‑апгрейда.

```
authorization: {
    timeout: 3
    users: [
        {user: a, password b},
        {user: b, password a}
    ]
}
```
