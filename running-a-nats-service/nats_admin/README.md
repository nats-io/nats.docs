# Управление и мониторинг инфраструктуры NATS Server

Управлять NATS Server просто. Типичные операции жизненного цикла включают:

* Использование CLI‑инструмента [`nats`](../../using-nats/nats-tools/nats_cli/) для проверки связности и задержек в кластере серверов, получения информации об аккаунтах, а также управления и работы со стримами (и другими приложениями NATS). Попробуйте следующие примеры, чтобы изучить наиболее распространенные способы использования `nats`.
  * `nats cheat`
  * `nats cheat server`
  * `nats stream --help` — мониторинг, управление и работа со стримами
  * `nats consumer --help` — мониторинг и управление потребителями стримов
  * `nats context --help` — если нужно переключаться между серверами, кластерами или учетными данными пользователя
* Использование CLI‑инструмента [`nsc`](../../using-nats/nats-tools/nsc/) при JWT‑аутентификации и авторизации для создания и отзыва JWT и ключей операторов, аккаунтов и пользователей (то есть клиентских приложений).
* [Отправка сигналов](signals.md) серверу, чтобы перезагрузить конфигурацию или повернуть файлы логов.
* [Обновление](upgrading_cluster.md) сервера (или кластера).
* Понимание [медленных потребителей](slow_consumers.md).
* Мониторинг сервера через:
  * мониторинговый [endpoint](monitoring/) и инструменты вроде [nats-top](../../using-nats/nats-tools/nats_top/)
  * подписку на [системные события](../configuration/sys_accounts/)
* Корректное завершение работы сервера с помощью [Lame Duck Mode](lame_duck_mode.md)
